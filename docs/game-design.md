# 《接住小团子》游戏设计文档

| 项目 | 内容 |
|------|------|
| 游戏名称 | 接住小团子 (Catch the Puppy) |
| 游戏类型 | 反应节奏类休闲小游戏 |

## 🎮 核心玩法

### 游戏流程
```
开始游戏 → 小狗从顶部掉落 → 玩家看准时机点击 → 接住 (得分+1) 
→ 下一只小狗掉落 (速度加快) → 循环 → 漏接 → 游戏结束 
→ 显示分数/排名 → 分享复活/重新开始
```

### 操作方式
- **单指点击**：屏幕任意位置点击
- **判定机制**：小狗进入底部判定圈时点击 → 成功接住
- **连击系统**：连续接住触发连击特效和额外加分

### 得分规则
| 行为 | 得分 |
|------|------|
| 普通接住 | +1 分 |
| 完美时机（判定圈中心 1/4 区域） | +2 分 |
| 连击 bonus（每 5 连击） | +3 分 |
| 关卡奖励 | 每关 +10 分 |

### 干扰元素

| 干扰物 | 解锁关卡 | 效果 | 应对策略 |
|--------|----------|------|----------|
| **树叶** 🍃 | 第 2 关 | 飘落干扰，点击打断连击 | 忽略树叶，专注小狗 |
| **假动作** 🎭 | 第 3 关 | 小狗中途停顿 0.5 秒 | 等待小狗重新开始下落 |
| **分身** 👻 | 第 4 关 | 产生半透明假影 | 点击中间的小狗（真身） |

## 📈 难度曲线（关卡制）

### 关卡设计

| 关卡 | 分数要求 | 下落速度 | 生成间隔 | 干扰元素 | 特殊机制 |
|------|----------|----------|----------|----------|----------|
| **第 1 关** 🌱 | 0-9 分 | 4 px/f | 1200ms | 无 | 教学关 |
| **第 2 关** 🍃 | 10-19 分 | 6 px/f | 1000ms | 树叶 | 随机飘落干扰 |
| **第 3 关** 🎭 | 20-29 分 | 7 px/f | 900ms | 树叶 + 假动作 | 小狗停顿欺骗 |
| **第 4 关** 👻 | 30-39 分 | 9 px/f | 700ms | 树叶 + 假动作 + 分身 | 双影迷惑 |
| **第 5 关** ⚡ | 40-49 分 | 11 px/f | 500ms | 全部 | 快速模式 |
| **第 6 关** 🔥 | 50-69 分 | 13 px/f | 400ms | 全部 | 地狱难度 |
| **第 7 关** 👑 | 70 分+ | 15 px/f | 300ms | 全部 | 传说模式 |

### 关卡过渡
- 每 10 分进入下一关
- 过渡时有 1.5 秒休息时间（暂停生成）
- 显示关卡名称和特效

### 速度公式
```typescript
// 每关有固定速度配置，不再使用动态公式
const LEVEL_CONFIG = {
    1: { speed: 4, spawn: 1200 },
    2: { speed: 6, spawn: 1000 },
    // ...
}
```

## 🛠️ 技术架构

### 项目结构
```
catch-the-puppy/
├── assets/
│   ├── scripts/
│   │   ├── GameManager.ts      # 游戏主控制器
│   │   ├── Puppy.ts            # 小狗逻辑
│   │   ├── TargetCircle.ts     # 判定圈逻辑
│   │   ├── UIManager.ts        # UI 管理
│   │   ├── AudioController.ts  # 音频管理
│   │   └── AdManager.ts        # 广告管理
│   ├── images/                 # 图片资源
│   ├── audio/                  # 音频资源
│   └── fonts/                  # 字体资源
├── settings/
│   └── project.json            # 项目配置
└── docs/                       # 设计文档
```

### 核心类设计

#### GameManager（游戏主控制器）
```typescript
- 管理游戏状态（menu/playing/gameover）
- 控制小狗生成节奏
- 计算分数和连击
- 处理游戏结束逻辑
- 接入微信排行榜
```

#### Puppy（小狗）
```typescript
- 下落动画控制
- 速度计算（根据难度）
- 碰撞检测
- 假动作逻辑（高阶难度）
```

#### AdManager（广告管理）
```typescript
- 激励视频广告接入
- 复活功能
- 解锁皮肤功能
- 广告冷却时间管理
```

## 💰 商业化设计

### 广告点位
| 场景 | 广告类型 | 触发条件 | 奖励 |
|------|----------|----------|------|
| 复活 | 激励视频 | 游戏结束后点击复活 | 继续游戏，保留分数 |
| 解锁皮肤 | 激励视频 | 点击锁定皮肤 | 永久解锁新小狗皮肤 |
| 双倍积分 | 激励视频 | 游戏开始前选择 | 本局得分 x2 |
| 每日奖励 | 激励视频 | 每日首次登录 | 随机道具/积分 |

### 广告频率控制
- 复活：每日限 3 次
- 解锁皮肤：每个皮肤仅需观看 1 次
- 避免强制插屏广告（影响体验）

## ⚠️ 风险提示

1. **版号问题**：休闲小游戏可走"个人开发者"通道，无需版号
2. **审核风险**：避免敏感内容，广告点位需符合微信规范
3. **性能优化**：低端机型帧率保障（目标 60fps）
4. **竞品差异化**：治愈系画风 + 小狗 IP 是核心差异点
